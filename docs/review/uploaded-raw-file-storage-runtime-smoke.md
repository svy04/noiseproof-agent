# Uploaded Raw File Storage Runtime Smoke

Phase marker: uploaded raw file storage runtime smoke v0.

Status: local runtime evidence.

This smoke records Docker PostgreSQL plus live FastAPI HTTP evidence for the quarantined raw upload storage boundary added in Phase 247.

This is local runtime evidence only, not hosted deployment evidence.

## Source-first anchors

- FastAPI file upload handling: https://fastapi.tiangolo.com/tutorial/request-files/
- OWASP File Upload Cheat Sheet: https://cheatsheetseries.owasp.org/cheatsheets/File_Upload_Cheat_Sheet.html

Interpretation used for this smoke:

```text
UploadFile provides the multipart boundary.
The original filename must stay metadata-only.
The storage key must be generated by the service.
Stored raw bytes must not be returned by metadata routes.
Oversized inputs must fail before persistence.
```

The local Docker volume already contained earlier raw-upload rows from prior smokes. This proof claim is limited to the latest row matching the POST response id.

## Behavior Observed

The smoke observed:

```text
raw bytes can be stored in PostgreSQL BYTEA
metadata routes return content hash, size, storage backend, storage key, and quarantine boundary
metadata routes do not return raw_bytes
the generated storage key does not contain the original filename
uploads larger than max_raw_upload_bytes are rejected with HTTP 413
```
