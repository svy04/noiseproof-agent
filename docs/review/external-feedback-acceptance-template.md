# External Feedback Acceptance Template

Status: template for future manual acceptance.

Phase marker: external feedback acceptance template v0.

Label: External feedback acceptance template.

## Purpose

Use this template only after issue #1 receives a public outside comment that passes the screening workflow and satisfies `docs/review/external-feedback-intake-criteria.md`.

Use `docs/review/external-feedback-acceptance-draft-cli.md` to produce a draft from a candidate screening artifact before manually filling this template.

This template exists so accepted external feedback can be recorded consistently without mixing request infrastructure, bot output, generic praise, or self-authored notes into the proof path.

## Template

### Reviewer identity

```text
Reviewer:
Reviewer role:
Reviewer relationship to repository owner:
Public comment URL:
Comment date:
```

### Inspected evidence

List the artifacts the reviewer explicitly inspected or referenced.

```text
- Artifact:
  Reviewer reference:
```

### Accepted critique

Record only actionable critique that names evidence, missing evidence, overstated claims, implementation gaps, or reviewer-path friction.

```text
Critique:
Why it qualifies:
Related claim boundary:
Required follow-up:
```

### Manual acceptance decision

```text
Accepted as external reviewer feedback: yes/no
Accepted by:
Acceptance date:
Reason:
```

### Claim boundary update

```text
Allowed public claim after acceptance:
Forbidden public claim after acceptance:
Next evidence gate:
```

## Boundary

This template is not external reviewer feedback.

It is not proof that any reviewer has inspected the repository.

It is not customer validation.

It is not Braincrew acceptance.

It is not hosted deployment evidence.

It must not be filled with self-authored feedback, generic praise, empty acknowledgement, CI output, bot summaries, or private unverified messages.
